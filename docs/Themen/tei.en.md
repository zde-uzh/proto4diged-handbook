# TEI model

The established standard for labelling digital editions is TEI/XML (Text Encoding Initiative).
However, there is disagreement on how the individual tags should be used.

The best starting point is when a parallel project exists whose schema can be transferred to the new project.
Other points of reference are the [DTA basic format](https://www.deutschestextarchiv.de/doku/basisformat/index.html) or [TEI simple](https://teic.github.io/TEI-Simple/).

## P5 ODD

For TEI conformity, it is not enough to simply use elements in the TEI namespace; an [ODD](https://tei-c.org/guidelines/customization/getting-started-with-p5-odds/){:target="\_blank"} must also be written that documents the use and extensions.
This goes beyond the definition of styles, as is standard with TEI Publisher.
