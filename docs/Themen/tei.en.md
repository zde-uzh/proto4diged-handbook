# TEI model

The established standard for encoding digital editions is TEI/XML (Text Encoding Initiative). However, there is disagreement about how the individual tags should be used in practice.

The best starting point is when a similar project already exists, whose schema can be adapted for the new project.
Other points of reference are the [DTA basic format](https://www.deutschestextarchiv.de/doku/basisformat/index.html) or [TEI simple](https://teic.github.io/TEI-Simple/).

## P5 ODD

TEI compliance is not just about using elements from the TEI namespace. To achieve full TEI conformity, an [ODD](https://tei-c.org/guidelines/customization/getting-started-with-p5-odds/){:target="\_blank"} s required. This ODD document outlines:
- How elements are to be used.
- Any customizations or extensions that have been made to the TEI standard.
An ODD goes beyond just defining styles and structural rules.
